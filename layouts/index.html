What is the JAMstack?

It’s a way of building a site so that it doesn't rely on any web servers or data bases. JAMstack sites are static and pre-built, meaning that they can be served directly from a Content Delivery Network.

Benefits of JAMstack

Speed 🏎

Because the sites are static and pre-rendered, loading times are much faster compared to dynamic sites, which require the server to rebuild the site each time it’s opened.

Security 🔒

There are no data bases or servers, therefore there is a much lower chance of a security breach.

Ease 👌

Since there are no servers or data bases, you don’t need to worry about setting them up. Front-end developers no longer need to mess around with the back-end. It’s a much better development experience!

Cheap 💵

Again, ditching servers and data bases means you don’t have to pay for them. Hosting can also be free on CDN services such as Netlify.

A JAMstack site has a static architecture

This does not mean that the user experience has to be static!

Here's what the JAM stands for

JavaScript

JavaScript is used to add interactivity and other bells and whistles to the site. Since there are no servers, the browser serves as the runtime.

APIs

Need to pass in data? No problem. APIs are used for external data access.

Markup

Markup files such as Markdown are source files from which the static HTML files can be generated from.

!

These technologies enable developers to achieve a dynamic experience on a static website. Keep in mind that a JAMstack site does not need to use all of these. For example, your website doesn’t need to have any JavaScript, it’s optional.

Ready to make your own JAMstack site?

Let's get started 🚀

  1. Create a static website

    The first thing you need is a static website. This can be handwritten from the ground up, using a static site generator, or both. The point is that the website is static so that it can be pre-rendered by a CDN.

    I recommend using a Static Site Generator such as Hugo as it adds tons of functionality to your website, and allows you to integrate a CMS into your website later.

    You can discover many more Static Site Generators on staticgen.com. Some are simple and only require simple HTML and CSS skills to get going, and some make use of JavaScript frameworks like React.

  2. Upload your site to a Git repository

    The CDN will need access to your website so that it can build it. This step can be done at the beginning of creating your website to make use of version control, but it’s not necessary.

    Once you are ready to publish the site with the latest changes, create a new Git repository on a website such as GitHub, then push the files to the repo. If you already have a repo set up for the site, simply push the latest changes.

  3. Link your Git repo to a CDN

    In the CDN service of your choice, create an account and import your website by connecting the Git repository. I recommend using Netlify as it’s very simple and free.

    Steps on Netlify:

    1. Once you have logged into your account, on the ‘Sites’ tab you will see a green button with the ‘New site from Git’ label. Press it to add your website from its Git repo.

    2. Sign in to your Git provider’s account when prompted.

    3. Netlify will now display the available repositories. Pick the repository of the website that you want to publish.

    4. Now you will be able to pick the branch you want Netlify to deploy your site from and the basic build settings. For example, Hugo uses the ‘hugo’ command to build the website, therefore the ‘Build Command’ setting will be set to ‘hugo’. You can also name the folder you want the site to publish into.

      Netlify works with some SSGs by default, therefore if you use Hugo, these settings will automatically be filled in for you.

    5. Press the green ‘Deploy site’ button. Congratulations! Netlify will now run the build task and publish your website. If everything went well, your website will now be live.

    You can now change the name of the website, which will also change the URL. You don’t need to do anything else from here. Every time a change is pushed to the Git repo of your site, Netlify will automatically rebuild and deploy it.

Thats it! 🎉

You now have a JAMstack website

CMS Integration (Optional)

If you’re interested in adding a content management system to your website (and I recommend that you do, especially for blog sites), it’s a very simple process thanks to services like Forestry CMS.

This is what the process of integrating Forestry into a Hugo site looks like:

  1. On the homepage of Forestry.io, press the ‘Import Your Site Now’ button, then create an account.

  2. On the ‘My sites’ page, press the green ‘Add site’ button.

  3. Choose the static site generator that your website uses and the version then press next.

  4. Choose the Git provider you use and make sure that Quick Setup is checked on then press next.

  5. Sign in to your Git account then select the repository and branch of your site then press next.

Forestry will now import your website and in a few short moments you site will have CMS functionality! You can now edit or add new pages amongst many other things. The steps to set this up were just as short and easy as linking a site with Netlify.

You can find a list of CMSs on headlesscms.org.
